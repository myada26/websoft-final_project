## Revised: FR-0023 · Digital Receipts *(enhanced)*

> **Actor:** System (auto-generated on every completed transaction)

- [ ] A PDF receipt is generated for every completed transaction, immediately upon confirmation
- [ ] Upon successful transaction, the system **automatically sends the PDF receipt to the student's institutional email** address on file
- [ ] If no email is on file, receipt is still generated — officer is prompted to print a physical copy instead
- [ ] Email subject format: `Official Receipt [OR Number] — [Organization Name] · [Semester]`

**Receipt must include:**
- University name and organization name (College Council / Dept Society)
- OR number in format `{ORG_CODE}-{PADDED_NUMBER}` (e.g., `CSCo-00237`)
- Student name, student number, program, year level
- Transaction type (`Membership Fee` or `Absence Fine`)
- Fee category (Regular / Irregular / Extendee / Exempted) — for Membership Fee only
- For Fines: events covered, slots missed, fine rate per event
- Total amount due, previous payments, amount paid this receipt, balance remaining
- Payment status: **✓ FULLY PAID** or **⏳ PARTIAL PAYMENT**
- Payment method (Cash / GCash) and GCash reference number if applicable
- Processing officer name and role
- Date, time, and active semester
- Footer warning if partial: *"Remaining balance of ₱X.XX must be settled before end of semester."*

**Format:**
- [ ] Formatted for **thermal printer (80mm/58mm)** or **A4 half-sheet**

## Revised: FR-0024 · Summary of Receipts (SOR) Report

> **Actor:** Treasurer *(own organization)* · SSC Admin *(cross-organization)*

**Core Concept:**
The Summary of Receipts is the official financial collection document submitted to the Office of Student Affairs (OSA). It is organized by OR batch range and categorized by transaction type. The Treasurer exports their organization's SOR; the SSC Admin can export across all organizations for cross-checking prior to submission.

**SOR breakdown columns:**

| Column | Description |
|---|---|
| OR Range | Batch of 50 receipts (e.g., 001–050) |
| Regular | Count of regular membership fee payments |
| Irregular | Count of irregular rate payments |
| Extendee | Count of extendee rate payments |
| Partial | Count of partial payments with outstanding balance |
| Fines | Count of absence fine payments |
| Voided | Count of voided/cancelled receipts |
| Count | Total receipts in batch |
| Subtotal | Total peso amount collected in batch |

**On-screen report includes:**
- [ ] Header: organization name, semester, total members, OR series range
- [ ] Stat cards: total receipts issued, total collected, count per category
- [ ] Collection breakdown by type showing per-unit rate and total per category
- [ ] OR batch table with subtotals and grand total row
- [ ] Signature blocks: Prepared by (Treasurer) · Checked by (OSA Representative)
- [ ] Export PDF button (Treasurer and SSC Admin only)

**Print / Export document includes:**
- [ ] University name: **Central Mindanao University · Office of Student Affairs**
- [ ] Organization name, academic year, semester, total members, date prepared
- [ ] Fee rates per category (Regular, Irregular, Extendee)
- [ ] OR series range (e.g., CSCo-00001 – CSCo-00539)
- [ ] Summary cards: Receipts, Total Collected, Regular, Irregular, Extendee, Partial
- [ ] Batch-by-batch table with type pills per row and batch subtotals
- [ ] Grand total row
- [ ] Three signature lines: Treasurer · OSA Representative · Adviser / Dean
- [ ] System-generated footer: *"Generated by FCATS · Central Mindanao University · Printed on ___"*

**Access control:**
- [ ] **Treasurer** — can view and export SOR for their own organization only
- [ ] **SSC Admin** — can view and export SOR for any or all organizations (cross-org)
- [ ] **Chairperson, Auditor, Collectors** — can view on-screen report but **cannot export**

## New: Partial Payment Feature

> **Applies to:** FR-0015 (Dynamic Assessment) · FR-0016 (Payment Methods) · FR-0017 (Single-Item Transactions)

**Core Concept:**
Partial payment accommodates students who cannot pay their full membership fee in one visit. Each partial payment generates its own OR number and receipt. The system tracks the running balance per student per semester per organization until fully settled.

**POS flow:**

```
Officer selects student → selects category: Regular (₱1,000.00)

  System displays:
    Total Amount Due:         ₱1,000.00
    Previously Paid:          ₱500.00   ← from prior partial OR, if any
    Remaining Balance:        ₱500.00

  Officer enters amount to collect: ₱300.00
    → System validates: amount > 0 AND amount ≤ remaining balance
    → If amount < remaining balance:
         payment_status = PARTIAL
         balance_remaining = ₱200.00
    → If amount = remaining balance:
         payment_status = FULL
         balance_remaining = ₱0.00
    → Receipt generated with this OR's paid amount and remaining balance
```

**Constraints:**
- [ ] Partial payments are allowed for **Membership Fee transactions only**
- [ ] Fines must be paid in **full** — no partial fine payments (FR-0013)
- [ ] Each partial payment generates a **separate OR number** and receipt
- [ ] The system accumulates all partial payments per student, per semester, per organization
- [ ] Remaining balance is always **system-computed** — officers cannot override it
- [ ] Students with outstanding partial balances are flagged in the SOR under the **Partial** column
- [ ] Chairperson can view all students with partial balances in Transaction History

**`TRANSACTIONS` table additions:**

| Column | Type | Notes |
|---|---|---|
| `payment_status` | Enum | `FULL` · `PARTIAL` |
| `balance_remaining` | Decimal(10,2) | `0.00` if FULL; outstanding amount if PARTIAL |

## Report Tab Structure (UI)

The **Reports** section is accessible from the main navigation bar. It contains three sub-tabs:

| Tab | Status | Description |
|---|---|---|
| **Summary of Receipts** | ✅ Active | SOR with on-screen report and print/export view |
| **Logs** | 🔲 Placeholder | Audit log viewer — future implementation |
| **Liquidation** | 🔲 Placeholder | Liquidation report for OSA — future implementation |

The on-screen SOR view has two inner tabs:
- **On-screen report** — interactive stat grid, breakdown cards, batch table, signature blocks
- **Print document** — A4-formatted document preview ready for PDF export via DomPDF

The receipt preview panel (right sidebar) shows three switchable receipt types:
- **Fully Paid** — standard complete payment receipt
- **Partial** — partial payment receipt with balance remaining and warning footer
- **Fine** — absence fine payment receipt showing events, slots missed, and fine rate
